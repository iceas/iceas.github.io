2018-11-27 /

Free 更新日志 3.0.1

2018/11/19 - 2018/11/24: 3.0.1.0

新增：

食物提供的能量按照三种能源物质计算，有致癌物项。饱食度机制重构。

加入动作的概念，单位时间耗能由动作种类影响。

加入健康系统，影响做事效率，以及接下来生病的概率、战斗能力。

调整：

一些动作所需的时间具有随机性。

加入生命自动恢复，单位时间回复量与血量定为二次函数关系。未积分。

背包函数改写，支持手动移动物品，选中物品时该物品的边框四角改变。支持多个格子放置同一物品。

打字机同时显示下一个将要提供的单词。

饱食状态新增：腹胀。一不小心吃了太多，直接威胁生命。

定义了一些难记的代号。

精简了关于睡意的状态。

添加一些音效。

一些地图改变。

优化音频。

状态栏闪烁可设置关闭。

修改了一些函数的类型。

主界面BGM改成钟琴。

重构了状态输出判定。

修改了大量常数、语言。

工程结构改变。

函数修复和优化。

以上是 3.0.0.17 => 3.0.1.0。你可能不介意了解 3.0.0 的更早的版本更新。

2018/11/17: 3.0.0.17

无效动作的提示信息：连续的胡思乱想。

2018/11/16: 3.0.0.16

语言的大幅修改。语言短小，更要尽可能生动，提供充足的想象引导。可参考另一游戏生火间。

采用32bit编译器。(bilibili)

Meas.h改名为Inter.h。

2018/11/3:

修复询问饼干信息不耗时的bug。

增加了写字楼供应餐的能量：30.0 -> 34.0

新增物品：烤鸡，高能量的快餐。

修复源码中的大量细节问题。

尝试加入新地点：广场。

2018/11/2:

界面不再带有”提示”。

加入读入数字的标准光标定位函数 Pstint()。

细节修改。

按 h + q 查看快捷键时，显示下次自动保存的剩余步数。

2018/11/1:

修复bug，如打字室无法输入。

2018/10/31：

源码风格改变。

显示任务的方式更改。

2018/10/30: 3.0.0.15

用Dev C++的工程开发Free。

细节改动。如：主界面不再是一个地点；吃东西按照该物品是否可携带判断是不是在背包内，进而判断需不需要 Watchout()；Move() 函数中离开原位置将会输出Map上对应的符号，为将来的小岛做准备。

修复吃面包效果等同于饼干的bug。

2018/10/28:

支持用时间单位表示秒数。如表达式 1 days（需复数形式）意味着86400秒。

取消 using namespace std。

物品信息改为结构体常量。

常用功能函数加了 inline。

对于很多常量字符串，用 const char * 替代 string。

加入Hoare的任务4-1（到3-1）。

函数 Printl() 支持传入每个字符的延迟时间以及句子结束后的延迟时间，还支持按键跳过。

一些变量名称改得更易理解。

吃东西函数 Eat() 优化。

主界面更改了。主界面背景乐在修改。

之前用单个 getch() 来暂停，现在写了一个函数 Pause()，防止上下左右键输入一个额外的字符而不被清理。

修改一些细节。

2018/8/1:

大量细节改动。

2018/7/31:

细节改动。

修改状态变化、输出的方式。

2018/7/30:

细节改动。

完善包裹。包裹可持续化运作！

总体上，保持较饱的代价降低了。

2018/7/29: 3.0.0.14

修改了一些对话内容。

加入了包裹。

代码细节优化了。

2018/7/28:

修改了暂停规则。

修改了对话显示方式。

2018/7/27:

查看任务时，语句越长则等待时间越长。

使用了Dev C++ 5.9.2，仍然开启-Wall。此-Wall标准不同。

新增任务系列二。动物园地图随之修改。

恢复了打字室工作奖励餐的较高要求。

饥饿和困倦对生命的威胁不随生命上限的提高而增大。

2018/7/26:

降低了工作奖励餐的要求：底线时间 60s(现实) -> 45s(现实)

修改了卧室Map，可能更不容易出现bug。

生命充足时状态”快活”指向不明，修改为”活跃”。

饥饿和困倦对生命的威胁将随生命上限的提高而增大。

介于清醒和疲倦之间，在凌晨4时睡觉，平均仅睡2h，似乎睡眠时间太短了。现修改睡意中间状态下的睡眠时间：预期至下一个早晨6时 -> 4h + (距离下一个早晨6时的时间) / 2

不再通货膨胀了。

超市购买提示中添加实际支出。

2018/7/25 - 2018/7/26: 3.0.0.13

重新设置初始界面的背景乐。

初始界面添加了大写提示。

修改了一些语言细节。

添加了超市的帮助。

修复了输入数字后再次输入数字出现的无法正常输入的问题。

修复了新建角色时设置无效的问题。

“新建角色”时初始属性变化：饱食度 60.0 -> 48.0 由于任务1-2的设定是买面包，应当体现”小饿”。

添加了打字系统的启动时间：320s，打击通过[退出后立即进入]、[打开帮助]获取高奖金。

初始界面的”Free”图形改变了。

减缓了饥饿时的血量流失。

自定义变量的名称去除了下划线。

2018/7/24 - 2018/7/25: 3.0.0.10 - 3.0.0.11 - 3.0.0.12

添加了颜色提示的开关colorOpen，并加入设置。新建角色，将会首先进入设置。进入帮助后也可以设置。

尝试让读档兼容先前的版本。

现在提示框MessageBox会显示在最上层了。

修改了一些语言、地图的细节。

修改了一些关于进食的内容。

修改读入，读入过大的数字不会退出游戏。

修改了一些帮助内容。拒绝卧室内进入帮助。

加入了困倦机制。

可以到卧室睡觉了。

修复了一些问题。

加入了任务系统。

2018/7/23: 3.0.0.9

读入string和int时，自动显示光标。

支持上下左右键移动。

加入保存系统。

加入更多音效。

现在打字室每一步都会增加总工分，避免打字室退出并读档后所赚的工分丢失。

源码中加入注释书签。Dev编辑时，Ctrl + F 到搜索文件内容，输入”mf” + 后缀（state, file等，可以不管大小写）并搜索，可以快速来到指定位置。例：输入”mfstate”，来到状态处理的地方。

2018/7/22:

加入更多音效，实现了背景音乐。

加入状态机制。

尝试加入进食。

2018/7/21:

加入一些音效，尝试加入背景音乐。

防止“利用[帮助]在打字室中跳过单词，却不影响APM”。

暂时屏蔽了符号颜色变化的机制。

卧室开始自信起来。

2018/7/17:

正式加入动物园。

2018/7/14:

重构场景切换代码，统一在Gate()（空间传送门）切换。

正式加入时间系统。尝试：符号颜色随时间推移而变化。

2018/7/13:

打字室结束时，将统计基础工资。

修改打字室计时方式。开启帮助不会计时。

正式加入银行。

2018/7/12: 3.0.0.3

Dilute:

加入打字室地图，以免未来无法显示提示和状态。

恢复了原APM计算方法。

Hoare:

打字室退出时不再减少工分。

xuxing:

在主地图中加入银行；绘制Bank.in。

Hoale:

卧室输出中文再现问题，置入修复。

用黑科技禁止exe窗口大小改变；更改窗口标题。

[人工扫地式检查]：

在源代码加入了更多注释。

统一：单字符、字符串、数字的输入函数，都到Read(int c)中，c代表类型。1为char，2为string，3为int。

改动：一些参数的名称，似乎更合理了；一些地图；提高了奖励餐的要求。

修复：一些问题。

添加：文件读取错误用MessageBox提示。

2018/7/9:

这几天大家的努力我看到了！因此直接拿Dilute下午上传的文件夹作为终端了。

然后我进行了一些修改，稍微平衡一下。

1.Supermarket，M不再大写。

2.所有撞墙情况，用一个Knock()函数判断。

3.修改了地图的细节。现在所有出入口都用”X”表示了。密室的出入口，在txt文件里是’!’，游戏中自动显示空格，那样就不用特判了。

出入商店（非移动的场景），按钮不是0了，是进入的反方向，比如向左走进超市，按d走出超市。

4.地点代号、函数目录在《编写须知》。

5.添加了延时输出的提示方法，在写字楼的无人办公室试验，体现探秘感。未来可以用于解谜。

6.一些语言修改。

7.现在所有 while(1) 都替换为 while(!Out)，医院设计者的想法！这更符合逻辑。

8.所有地图的坐标已经采用全局变量，而地点代号也是一个全局变量。为什么呢？

读档会更方便。

不用重复累赘地传入参数，存档会方便、合理很多。

9.连续的大门都变小了一些。

10.避免代码太长，移动步骤塞入Move函数。

等等

修改后，功能基本不变，代码长度缩短了10%！

后来啊，代码又增加了30%……

在7月10日，增加了打字工作和帮助内容。

为了随时随地帮助，超市等地方的地图输出与其他地方统一了。
